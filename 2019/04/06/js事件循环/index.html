js事件循环

发表于 2019-04-06 | 分类于 前端

菜鸡的js事件循环理解

javascript是单线程的语言，也就是说，同一个时间只能做一件事，原因之一在其最初也是最主要的执行环境——浏览器中，我们需要进行各种各样的dom操作。试想一下 如果javascript是多线程的，那么当两个线程同时对dom进行一项操作，例如一个向其添加事件，而另一个删除了这个dom，此时该如何处理呢？因此，为了保证不会 发生类似的情景，javascript选择只用一个主线程来执行代码，这样就保证了程序执行的一致性。

单线程就意味着，所有任务需要排队，前一个任务结束，才会执行后一个任务。如果前一个任务耗时很长，后一个任务就不得不一直等着。

但这对于浏览器的网络请求显然是不合适，因为一个网络请求的资源什么时候返回是不可预知的，这种情况再排队等待就不明智了

我们把任务分为同步任务和异步任务，两者区别在于是否需要等待结果返回

正是由于JavaScript是单线程的，而异步容易实现非阻塞，所以在JavaScript中对于耗时的操作或者时间不确定的操作，使用异步就成了必然的选择

函数调用栈与任务队列

js主线程运行时，形成一个执行栈，所有同步任务都在主线程上执行，当在执行过程中遇到异步操作时，会将它交给浏览器的其他模块进行处理，然后将它的回调函数放入任务队列中，不阻塞主线程的执行不同的异步任务的回调函数会放入不同的任务队列之中，等执行栈中的任务都执行完毕后，检查任务队列是否有任务要执行，如果有任务要执行的话，那么就将该任务放到执行栈中执行其中的同步代码。如果当前任务队列为空的话，它就会一直循环等待任务到来。因此，这叫做事件循环。

举例setTimeout：

1.执行第一句，将它放入执行栈中，输出1
2.第一句出栈，执行第二句，由于setTimeout是异步操作，将它交给对应模块，然后将回调函数放入任务队列中
3.执行第三句，入栈，输出3
4.第三句出栈，此时执行栈空了，检查任务队列，发现有回调函数，执行输出2

macro task与micro task

上文提到不同的异步任务回调函数会被放入不同的任务队列中
不同的异步任务被分为两类：微任务（micro task）和宏任务（macro task）
宏任务队列：

  1. script全局任务
  2. setTimeout //它在”任务队列”的尾部添加一个事件，因此要等到同步任务和”任务队列”现有的事件都处理完，才会得到执行。
  3. setInterval
  4. setImmediate（Node.js）
  5. I/O
  6. UI渲染

微任务队列：

  1. process.nextTick（Node.js） //一个特殊的任务，它会被直接插入到微任务的队首（当然了，多个process.nextTick 之间也是先入先出的），优先级最高
  2. Promise
  3. Object.observe
  4. MutationObserver

当添加了Promise后会怎样呢

考虑如下代码：

1.script全局任务执行，全局上下文入栈
2.遇到setTimeout,作为一个macro-task，将其回调函数放入宏任务队列中
3.遇到Promise实例。Promise构造函数中的第一个参数是在当前任务直接执行不会被放入队列之中，因此此时输出 1
4.在for循环里面遇到resolve函数，函数入栈执行之后出栈，此时Promise的状态变成Fulfilled。代码接着执行,输出2
5.代码遇到then方法，其回调函数作为micro-task入栈，进入Promise的微任务队列之中。
6.继续执行，输出3
7.输出3之后第一个宏任务script的代码执行完毕，这时候开始开始执行所有微任务队列之中的micro-task。then的回调函数入栈执行完毕之后出栈，这时候输出5
8.所有的micro-task执行完毕，第一轮循环结束。第二轮循环从setTimeout的任务队列开始，setTimeout的回调函数入栈执行完毕之后出栈，此时输出4。

总结

  1. 先执行macro-task，等到函数调用栈清空之后再执行所有在队列之中的micro-task。
  2. 等到所有micro-task执行完之后再从macro-task中的一个任务队列开始执行，就这样一直循环。
  3. 当有多个macro-task(micro-task)队列时，事件循环的顺序是按上文macro-task(micro-task)的分类中书写的顺序执行的。
  4. 一开始js主线程中跑的任务就是macrotask任务，而根据事件循环的流程，一次事件循环只会执行一个macrotask任务，因此，执行完主线程的代码后，它就去从microtask队列里取队首任务来执行。

最后考虑如下复杂代码：

  1. 宏任务队列中放入了一个setTimeout函数
  2. 输出了main1，promise 1，main2，Hello！
  3. 微任务队列中已经放入了promise then，nextTick3，nextTick4，nextTick2

10.所有任务队列清空
